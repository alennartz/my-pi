# DR-001: Codemap as Primary Agent Context

## Status
Accepted

## Context
Agents lack codebase context when planning and implementing, leading to hallucinated structures and missed constraints. Needed a way to give the agent a persistent, context-efficient understanding of the codebase.

## Decision
Use a living markdown file (`codemap.md`) at repo root as the primary context backbone. Auto-generated by the agent, human-editable. Each module section is self-contained with purpose, responsibilities, dependencies, and file globs. Mermaid diagrams for relationships. The agent loads it whole as a codebase guide before doing any targeted work.

## Consequences
All skills that need codebase awareness read the same artifact via a known path convention. The codemap must stay current — but updates are cheap since the agent rereads the full codemap before writing changes. Optimized for agent consumption (compact, information-dense) which also happens to be human-readable.
